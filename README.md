## Project 1 — Green Logistics: Carbon Footprint for E-Commerce

### Dataset
`Q1_logistics_carbon_emissions.csv` — 12,000 rows of logistics trip data including vehicle type, distance, package weight, traffic conditions, and carbon emissions.

### Question 1A — ETL Pipeline

**1. Database**
- Creates a `CarbonEmission` database in MariaDB using Python and SQLAlchemy

**2. Extract**
- Loads the CSV into a pandas DataFrame
- Replaces all dashes in column headers and `Route_Type` values with underscores
- Saves the raw data into a `combined` table in MariaDB

**3. Transform**
- Reclassifies `Mixed Route` entries based on distance:
  - `> 100 km` → `Inter_City`
  - `20–100 km` → `Inter_District`
  - `< 20 km` → `Urban Last Mile`
- Recalculates `Carbon_Emission_kgCO2e` using the formula:

```
Emission = (Base Factor × Package Weight × Traffic Multiplier) × Distance
```

| Vehicle Type | Base Factor | Traffic Condition | Multiplier |
|---|---|---|---|
| Cargo Bicycle | 0 | Normal | 1 |
| Drone Delivery | 0.02 | Low | 1.1 |
| Electric Van (EV) | 0.05 | High | 1.5 |
| Motorcycle (Courier) | 0.1 | Severe Congestion | 1.75 |
| Diesel Van (Euro 6) | 0.2 | | |
| Diesel Van (Euro 4) | 0.4 | | |
| Heavy Truck | 1.2 | | |

**4. Load**
- Creates 3 separate tables in MariaDB (`inter_city`, `inter_district`, `urban_last_mile`)
- Each table contains only its corresponding route type rows, ordered by vehicle type

### Question 1B — Inference & Insights
SQL queries on the `combined` table:
- Total emissions per vehicle type per route type
- Routes ranked by total distance (descending)
- Eco-friendly vs non-eco-friendly total distance for the year
- Total emission per vehicle type per month

### Question 1C — Visualization
- **Chart 1:** Histogram — distribution of carbon emissions per route type (log scale)
- **Chart 2:** Pie chart — routes ranked by total distance
- **Chart 3:** Monthly column charts — total emission per vehicle type per month (log scale)

## Project 2 — Streaming Data: eCommerce

### Dataset
`Online_Retail.xlsx` — 541,909 rows of UK-based online retail transactions from 2010–2011 (Chen, 2015). Available at [UCI ML Repository](https://doi.org/10.24432/C5BW33).

### Question 2A — Preprocessing
- Reads the dataset using pandas
- Maps each country to a region (Europe, Asia, Middle\_East, Americas, Africa, Oceania, Unspecified)
- Creates one DataFrame per region
- Saves each regional DataFrame as a CSV file

### Question 2B — Kafka Streaming
- **Generator function:** reads regional CSVs and yields one row at a time
- **Topics:** 7 Kafka topics created — one per region
- **Producer:** reads from the generator and routes each message to the correct topic based on the `Region` column
- **Consumer:** subscribes to all 7 topics and tracks:
  - Message count per region
  - Cumulative sales (`Quantity × UnitPrice`) per region
  - Unknown customers (`CustomerID = NaN`) per region
- **Summary table:** displays all tracked metrics per region

### Question 2C — Inference & Insights (Python + SQL)
Creates an `OnlineRetail` database in MariaDB and runs SQL queries:
1. Top 3 countries by number of purchases
2. Total sales for each of the top 3 countries
3. Most sold stock item per region (using `ROW_NUMBER()` window function)

### Question 2D — Visualization
- **Chart 1:** 3-panel bar chart — Kafka consumer summary (messages, sales, unknown customers) per region using log scale
- **Chart 2:** Bar chart — top 3 countries by number of purchases
- **Chart 3:** Dual y-axis bar chart — purchases and total sales for top 3 countries
- **Chart 4:** Horizontal bar chart — most sold stock item per region

## Key Results

### Project 1
- Heavy Trucks are the dominant emitter across all route types by a significant margin
- Inter\_City routes cover the most total distance
- Cargo Bicycles produce zero emissions by definition

### Project 2
- **United Kingdom** accounts for ~91% of all purchases in the dataset
- **Europe's** most sold item: WORLD WAR 2 GLIDERS ASSTD DESIGNS (52,742 units)
- **Asia's** most sold item: RABBIT NIGHT LIGHT (3,401 units)
- ~25% of all transactions have unknown CustomerIDs
